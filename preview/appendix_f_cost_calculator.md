A token is roughly the smallest unit of text that a language model processes. The practical rule of thumb is solid enough for budget estimation: **1 token ≈ 4 characters of English text** More concretely: | Content | Approximate Tokens | |---------|-------------------| | 1 word (English) | 1.3 tokens | | 1 sentence (15 words) | ~20 tokens | | 1 short paragraph (100 words) | ~130 tokens | | This page (500 words) | ~650 tokens | | A typical email | 300–500 tokens | | A page of code (50 lines) | 500–800 tokens | | A Python module (200 lines) | 2,000–3,500 tokens | | A chapter of this book | 8,000–15,000 tokens | **Why tokens matter for Claude Code specifically:** Every Claude Code session involves three categories of tokens: 1. **System tokens** — The system prompt: default Claude Code instructions, your CLAUDE.md, tool definitions, MCP descriptions. This is overhead before you type a single word. A default session starts at roughly 5,000–10,000 tokens. 2. **Context tokens** — The conversation history: all your messages, Claude's responses, tool call inputs, tool results. This grows with every turn. Reading a 500-line file costs ~2,500 input tokens. 3. **Output tokens** — What Claude generates: responses, code, explanations. These are more expensive per token than input on every model. **The asymmetry:** Output is 5x more expensive than input on Sonnet ($15 vs $3 per million). A single long code generation (5,000 output tokens) costs the same as five reads of a large file (5 × 2,000 input tokens). Code generation is expensive; reading is cheap. --- ## Worked Examples ### Example 1: The Typical Work Session **Scenario:** 90 minutes of focused development work. Over 30 turns, 29 reads hit cache: - Cache write (1 time): 8,000 × $3.75/M = $0.030 - Cache reads (29 times): 8,000 × 29 × $0.30/M = $0.070 - Without cache: 8,000 × 29 × $3.00/M = $0.696 - **Cache saves $0.626 on this one component alone** Adjusted total with cache: **~$0.22–$0.30** --- ### Example 2: Writing a 500-Line Python Module **Scenario:** Starting from scratch. Claude reads architecture docs and existing code, then writes a complete module. **Phase 1: Research** - Read 5 existing modules for context: 5 × 2,000 = 10,000 input tokens - Read architecture docs: 3,000 input tokens - Claude's planning response: 1,500 output tokens **Phase 2: Implementation** - 500 lines of Python at ~8 tokens/line: 4,000 output tokens - Docstrings and comments: +1,500 output tokens - Claude's explanation: 500 output tokens - Your review message: 100 input tokens **Phase 3: Review and iteration (2-3 rounds)** - Your feedback (3 × 150 tokens): 450 input tokens - Claude's revisions (3 × 1,500 tokens): 4,500 output tokens - Re-reading file after edits (3 × 2,000 tokens): 6,000 input tokens **Total: ~20,000 input tokens + ~12,000 output tokens** **Cost on Sonnet:** - Input: 20,000 × $3.00/M = $0.060 - Output: 12,000 × $15.00/M = $0.180 - **Total: ~$0.24** Writing a complete 500-line module costs roughly the same as two cups of coffee. The output tokens (code generation) are the dominant cost — 3× more expensive than all the reading combined. --- ### Example 3: Building a SaaS from Scratch **Scenario:** Six weeks of focused development. Building a complete product: database schema, backend API, frontend, tests, deployment. **Week 1: Architecture and scaffolding** - Exploration, planning, setting up structure - Input-heavy (lots of reading and back-and-forth) - Estimated: ~300K tokens **Weeks 2-4: Feature development** - Implementing endpoints, UI components, database queries - High output token usage (code generation) - Estimated: ~2M tokens per week × 3 weeks = 6M tokens **Weeks 5-6: Testing and polish** - Writing tests, fixing bugs, documentation - Mixed input/output - Estimated: ~600K tokens **Total estimate: ~7M tokens** Assuming 60% input ($3.00/M), 40% output ($15.00/M): - Input: 4.2M × $3.00/M = **$12.60** - Output: 2.8M × $15.00/M = **$42.00** - **Total: ~$55–$80 on Sonnet** With cache optimization, subtract 15-20%: **~$45–$65** The material cost of AI assistance to build a complete product from scratch is $50–$100. The AI tokens themselves are remarkably inexpensive relative to the value delivered. --- ### Example 4: The Rate Limit Hit Case **Scenario:** Power user, 5 hours of active Claude Code use every working day. **Usage estimation:** - 5 hours/day × 22 working days = 110 hours/month - Typical pace: ~100K tokens/hour - Monthly estimate: 110 × 100K = **11M tokens/month** **Breakdown (70% input, 30% output):** - Input: 7.7M × $3.00/M = $23.10/month - Output: 3.3M × $15.00/M = $49.50/month - **Pay-as-you-go: ~$73/month** At this level, Max Plan ($100/month) is slightly more expensive in pure token math. But rate limits change the picture: at 11M tokens spread across 110 hours, that's ~500K tokens/day. Concentrated sessions (6 intensive hours) can trigger rate limits even on Max Plan. The value of Max Plan here isn't cheaper tokens — it's predictable billing and reduced rate limit friction. --- ### Example 5: The Vibe Coder **Scenario:** Non-developer using Claude Code for weekend projects. Two sessions per week, 30 minutes each. Mostly reading and small file edits, occasional scripts. **Typical session:** - System prompt: 5,000 tokens (simple CLAUDE.md) - Your messages (5 exchanges × 100 tokens): 500 tokens - File reads (3 files × 1,500 tokens): 4,500 tokens - Claude responses (5 × 400 tokens): 2,000 output tokens - Small edits (2 × 500 tokens): 1,000 output tokens **Per session: ~10,000 input + ~3,000 output tokens** **Cost on Sonnet:** - Input: 10,000 × $3.00/M = $0.030 - Output: 3,000 × $15.00/M = $0.045 - **Per session: ~$0.08** **Monthly (8 sessions):** ~$0.64 **Max Plan break-even:** Never. Pay-as-you-go at roughly $1/month is obviously better than $100/month Max Plan. Free tier API access (if available) might cover this entirely. For occasional users, Max Plan is never the right choice economically — it only makes sense at sustained high-volume use. --- ### Example 6: The Startup Team **Scenario:** 5 developers using Claude Code as their primary development interface. **Option A: One Max Plan per developer** - 5 × $100/month = **$500/month** - Each developer gets rate limit protection - Predictable billing per seat - Simple: each developer manages their own plan **Option B: Shared API pool** - Total usage estimate: 5 developers × 11M tokens/month = 55M tokens - Blended rate (65% input, 35% output): ~$7.20/M effective rate - Monthly API cost: 55M × $7.20/M = **$396/month** - Variable billing — months with more development cost more - Requires a shared API key distribution mechanism **The break-even math:** At 55M tokens, Option B is cheaper by ~$100/month. But Option A has hidden value: - No "should I ask Claude?" hesitation (rate limits removed from the equation) - No billing complexity (each developer's usage is independent) - No shared-pool fairness problems (one developer's intensive sprint doesn't slow others) **The recommendation for most startups:** Max Plan per developer for the first year. The productivity benefit of zero usage anxiety outweighs the small cost difference. Revisit when you have usage data to optimize against. **Option C: Enterprise API agreement** For teams at 10+ developers, Anthropic offers volume pricing. Discounts typically kick in at sustained usage. Contact Anthropic's sales team; the per-token rate can drop significantly. --- ### Example 7: The Enterprise User **Scenario:** Large organization deploying Claude Code to engineering teams. Data residency, compliance, and volume are primary concerns. **AWS Bedrock pricing:** Bedrock re-sells Anthropic models with slightly different pricing and a data-residency guarantee. Approximate rates: - Sonnet: $3.00/M input, $15.00/M output (same as direct) - Plus AWS data transfer costs if your usage crosses availability zones **GCP Vertex AI pricing:** Similar structure to Bedrock. Models may be available with committed-use discounts for predictable workloads. **Why enterprises choose Bedrock/Vertex over direct:** - **Data residency:** Data stays in a specific AWS region / GCP region, satisfying compliance requirements - **Billing:** Unified billing with other cloud infrastructure (one invoice) - **IAM integration:** Access control via existing AWS/GCP identity management - **SLA:** Cloud provider's uptime SLA rather than Anthropic's **The math difference:** For pure API usage at scale, direct Anthropic pricing and Bedrock/Vertex pricing are similar. The enterprise premium is in the compliance and integration value, not the per-token rate. **Volume discounts:** At enterprise scale (tens of millions of tokens/month), both direct Anthropic and cloud providers offer negotiated rates. The general pattern: 10-30% discounts at meaningful volume. Model this as: your estimated monthly token spend × 0.75 for planning purposes. --- ## When Does Max Plan Pay Off? ### The Break-Even Calculation Max Plan costs $100/month.
